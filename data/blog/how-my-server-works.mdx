# Pendahuluan

Saat memikirkan tetang Deployment, orang-orang akan selalu berfikir tentang Hosting seperti VPS, AWS, GCP, Digital Ocean, Vercel dan lainnya. Tentu saja saya kerap kali menggunakannya. Namun, saya ingin sesuatu yang beda. Melakukan deployment di Cloud Hosting memanglah mudah dan cepat. Aplikasi kita dijamin untuk hidup 24/7 tanpa perlu khawatir listrik padam.

Di Level Production, saya selalu mempercayakan Cloud Hosting Service seperti yang sudah saya sebutkan diatas. Namun, di level development saya ingin melakukan suatu yang seru. Melakukan self deployment, yang benar-benar komputernya berada dirumah saya. Saya bisa melihatnya. Dan tentu saja semua bergantung pada kondisi kelistrikan dan internet ditempat saya berada. Saat internet gangguan, maka server saya juga akan gangguan. Saat Listrik padam maka artinya server/aplikasi saya tidak bisa kalian akses LMAO.

# Masalah

Rencana ini sebenarnya sudah lama saya ingin lakukan. Namun saya memiliki 2 masalah

1. Saya butuh komputer sever yang menyala 24/7, yang tentu akan sangat boros energi
2. Provider internet saya tidak menyediakan ip public.

Dua hal ini adalah hal yang wajib ada agar kita bisa meng-expose web kita.
